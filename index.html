Compare basic syntax of different programming languages. Currently supporting Python and C#

Basics

Syntax

Python is a dynamically-typed language, which means that it has typed objects but untyped variable names.

Python uses whitespace indentation, rather than curly brackets or keywords, to delimit blocks.

A Python script is run from top to bottom, so a function has to be declared before it is called.

Comments: Comments in Python start with a #

# This is a comment
print("Hello, World!") # This is another comment

Python does not really have a syntax for multi line comments. To add a multiline comment you could insert a # for each line:

# This is a comment
# written in
# more than just one line
print("Hello, World!")

Or, not quite as intended - since Python will ignore string literals that are not assigned to a variable, you can add a multiline string (triple quotes) in your code, and place your comment inside it:

"""
This is a comment
written in
more than just one line
"""
print("Hello, World!")

C# is a statically-typed language, which means that it has typed variables.

C# uses curly brackets to delimit blocks and semicolons to denote the end of a statement.

C# supports strongly typed implicit variable declarations with the keyword var. Typed variables allow us to not assign them any value:

int myNumber;
myNumber = 2;
var myVariable = "string";

The C# language does not allow for global variables or functions. All methods and members must be declared within classes. Static members of public classes can substitute for global variables and functions.

*/

Strings and printing into console

Printing text into console

print("Hello World!")
print('This works too')
print(451)
# Prints: >>>Hello World!
#         >>>This works too
#         >>>451

2 ways of printing "My name is John and I'm 25 years old." using variables:

name = "John"
age = 25

# Classic string concatenation
print("My name is "+name+" and I'm "+str(age)+" years old.")
# Variables must be strings - use type conversion function str()

# Efficient string formatting added in Python 3.6
print(f"My name is {name} and I'm {age} years old.")
# Note the automatic conversion of int age to string

Use of special character \n which means newline and a feature of string formatting useful for debugging. 

print(f"{name=}\n{age = }")
# Prints: >>>name='John'
#         >>>age = 25

Note that it keeps the spaces around equal sign we wrote in curly brackets.

Printing text into console

Console.WriteLine("Hello World!");
Console.Write("He said: ");
Console.Write("451");
Console.WriteLine("!");
// Prints: >>>Hello World!
//         >>>He said: 451!

Console.WriteLine adds a \n (newline) to the end of the string while Console.Write does not.

3 ways of printing "My name is John and I'm 25 years old." using variables:

string name = "John";
int age = 25;

// Classic string concatenation
Console.WriteLine("My name is "+name+" and I'm "+age+" years old.");

// String formatting
Console.WriteLine(String.Format("My name is {0} and I'm {1} years old.",
                                name, age)); // or
Console.WriteLine("My name is {0} and I'm {1} years old.", name, age);

// String interpolation
Console.WriteLine($"My name is {name} and I'm {age} years old.");

Note that all of these implicitly convert int age to string.

Types

None is Python equivalent to null in other languages. It's one of the falsy values and it's also the default value returned by functions that doesn't return anything, e.g. print().

Python booleans are written with upper first letter: True/False. true or false would raise an error.

my_boolean = True
my_boolean2 = 1 > 2 # = False

Create numeric variables of types: [int, float, complex] and assign them to 2.

my_integer = 2
my_float = 2.0
my_complex = 2j

Strings are iterable, immutable and ordered sequences. In Python, even characters like "c" are strings.

my_string2 = "" # Create empty string object
my_string2 = "bLue skY"
my_string2 = my_string2.upper() # = BLUE SKY
# .upper() method returns an uppercased copy of a string, 
# it doesn't change the string in-place!

Tuples are iterable, immutable and ordered sequences.

my_tuple = () # Create empty tuple object
my_tuple2 = (2, 2, 9.5, -4, "apple")
third = my_list2[2] # = 9.5; Accessing sequence item by it's index
my_tuple3 = ([5, 8], (8, 2))
my_tuple3[0][1] = 2 # my_tuple3 = ([5, 2], (8, 2))
my_tuple3[1][1] = 3 # --> Error (can't change tuples)

Lists are iterable, mutable and ordered sequences. Similar to arrays in other languages, but can hold different types.

my_list = [] # Create empty list object
my_list2 = [2, "Steve", ' ', True, 4.5, None]
item = my_list2[1] # Accessing sequence item by it's index
my_list2[2] = "third" # Changing sequence item by it's index
length = len(my_list2) # = 6

my_list2.append("last")
# Appends given object ("last") to the end of the list
my_list2.insert(-1, "second last")
# Inserts given object ("second last") BEFORE given index (-1)
my_list.extend(my_list2)
# Appends elements from given iterable (my_list2)

first = my_list.pop(0)
# Removes item at given index (0) and returns it's value --> first = 2
my_list.remove("Steve")
# Removes first occurence of given item (2) from list

# Sorting lists
# sorted() function returns a sorted list of items from an iterable.
# .sort() method changes original list and returns None.
# Ascending order on default, descending using reverse=True keyword.
unsorted_list = [8, -2, 84, 0, -99]
sorted_list = sorted(unsorted_list, reverse=True)
unsorted_list.sort(reverse=True)
# unsorted_list == sorted_list = [84, 8, 0, -2, -99]

Sets are iterable, mutable and unordered. They don't allow duplicates, which is useful for getting rid of duplicates from a list (if we don't mind losing order).

Note that empty set isn't declared using {} (that would be dictionary), but using set() function.

my_set = set() # Create empty set object
my_set2 = {2, 2, 9.5, -4, "apple"}
print(my_set2)
# Prints e.g.: {9.5, 2, 'apple', -4} - second 2 is gone, order is random

Dictionaries are iterable, mutable and ordered (from Python 3.7). A dictionary is basically a hash table in common terms.

my_dict = {} # Create empty dictionary object
my_dict2 = {"key": "value", "first": "John", 1: "John", 3.14: False}

my_dict2["new"] = "new value" # Adds a new key:value pair to the dictionary
my_dict2["key"] = "updated value" # Changes value with given key

first = my_dict2["first"] # first = "John"
first =  my_dict2.get("first") # first = "John"
second = my_dict2["second"] # --> Error (key "second" doesn't exist)
second = my_dict2.get("second", "Key not found")
# second = "Key not found", if left blank --> second = None

my_dict2.pop("key") # Removes a pair based on given key;
                    # Also returns the value (similar to .pop on list)
my_dict2.clear() # Removes all pairs

Python type conversion functions:

Boolean values are lowercase, default is false.

bool myBoolean = true;
bool myBoolean2 = 1 > 2; // = false
bool defaultBool = default(bool); // = false

Numeric data types include value range. The compiler will give an error if the value goes out of datatype's permitted range.

Type Description Range
byte 8-bit unsigned integer 0 to 255
short 16-bit signed integer -32,768 to 32,767
int 32-bit signed integer -2,147,483,648 to 2,147,483,647
long 64-bit signed integer long
float 32-bit Single-precision floating point type -3.402823e38 to 3.402823e38
double 64-bit double-precision floating point type long

There is also signed byte (sbyte), unsigned versions of others (uint, ulong...) and decimal for financial calculations.

The value of unsigned integers, long, float, double, and decimal type must be suffix by u, l, f, d, and m, respectively.

long number = 123L; // Upper L for better code readability
float number2 = 1.23f;
double number3 = 123d;

Strings are sequences of Unicode characters.

string myString = "bLue skY";
char myChar = myString[1]; // = 'L'
myString = myString.ToUpper(); // = BLUE SKY
// .ToUpper() method returns an uppercased copy of a string,
// it doesn't change the string in-place!

An array is the data structure that stores a fixed number of literal values of the same data type. It can be declared using the type of its elements with square brackets, new keyword, and a size.

int[] intArray; // Integer array
string[] cities = new string[3]; // String array of size 3
cities[0] = "Prague"; // Assigning value at index 0

// Declare and add values into an array in a single statement;
// Size is based on the number of elements inside curly braces
int[] evenNums = {2, 4, 6, 8, 10};
var evenNums2 = new int[]{2, 4, 6, 8, 10}; // or with var and new
var evenNumsInvalid = {2, 4, 6, 8, 10}; // --> Error

ArrayList is a collection of elements of the same or various data types (non-generic collection) with dynamic size. It has some useful methods not available with Arrays (e.g. Reverse, Contains, Sort). The class is in the System.Collections namespace.

using System.Collections;

ArrayList myArrayList = new ArrayList(); // Create empty ArrayList
var arList2 = new ArrayList(){2, "Steve", ' ', true, 4.5, null};
// An explicit casting to the appropriate types is required, 
// or use the var variable when accessing values
string item = (string)arList2[1];
var item2 = arList2[1];
arList2[2] = "third"; // Changing item by it's index
int length = arList2.Count; // = 6

arList2.Add("last"); // Add element to the end of the ArrayList
arList2.Insert(arList2.Count - 1, "second last");
myArrayList.AddRange(arList2);
// Add a collection (Array, ArrayList...) to the ArrayList

myArrayList.RemoveAt(0); // Removes item at given index
myArrayList.Remove(2); // Removes first occurence of given item

var unsortedList = new ArrayList(){8, -2, 84, 0, -99};
unsortedList.Sort(); // Sorts list in ascending order
unsortedList.Reverse();
// unsortedList = {84, 8, 0, -2, -99}

List is a collection of elements of the same data type (generic collection) with dynamic size. It has some useful methods like ArrayLists (e.g. Reverse, Contains, Sort). The class is in the System.Collections.Generic namespace. List performs faster and less error-prone than the ArrayList.

using System.Collections.Generic;

List<int> intList = new List<int>(); // Create empty List
var strList = new List<string>(){"Prague", "London", null};
string item1 = strList[0]; // No need for type casting
// ... (methods, assigning etc. similar to ArrayList)

Dictionary<int, string> numberNames = new Dictionaryint, string>();
var capitals = new Dictionary<string, string>(){{"UK", "London"},
                                                {"USA", "Washington"},
                                                {"India", "Pune"}};

numberNames.Add(1, "one"); // Adds a new key:value pair to the dictionary
capitals["India"] = "New Delhi"; // Changes value with given key

string USA = capitals["USA"]; // = "Washington"
string Germany = capitals["Germany"]; // --> Error (key "Germany" doesn't exist)
// Use .ContainsKey() to check for an unknown key
if (capitals.ContainsKey("Germany")){
    Console.WriteLine(capitals["Germany"]);
}

capitals.Remove("UK"); // Removes a pair based on given key
capitals.Clear(); // Removes all pairs

Logical, relational

In Python, individual values can evaluate to either True or False. Values that evaluate to False are considered Falsy, values that evaluate to True are considered Truthy.

Falsy values:
  1. Empty sequences and collections:
    • Empty lists []
    • Empty tuples ()
    • Empty dictionaries {}
    • Empty sets set()
    • Empty strings ""
    • Empty ranges range(0)
  2. Zero of any numeric type:
    • Integer 0
    • Float 0.0
    • Complex 0j
  3. Constants:
    • None
    • False
Truthy values:
  • Non-empty sequences or collections (lists, tuples, strings, dictionaries, sets)
  • Numeric values that are not zero
  • Constant True
none = None
zero = 0

if 1 < 2 and not zero or none:
    print("True")
else:
    print("False")
# Prints: >>>True

some_list = ["apple", "pear", "orange"]
if "apple" not in some_list:
    print(1)
elif zero != 0 and "orange" in some_list or len(some_list) > 2:
    print(2)

In Python, individual values can evaluate to either True or False. Values that evaluate to False are considered Falsy, values that evaluate to True are considered Truthy.

Falsy values:
  1. Empty sequences and collections:
    • Empty lists []
    • Empty tuples ()
    • Empty dictionaries {}
    • Empty sets set()
    • Empty strings ""
    • Empty ranges range(0)
  2. Zero of any numeric type:
    • Integer 0
    • Float 0.0
    • Complex 0j
  3. Constants:
    • None
    • False
Truthy values:
  • Non-empty sequences or collections (lists, tuples, strings, dictionaries, sets)
  • Numeric values that are not zero
  • Constant True

OOP

Python is a multi-paradigm programming language. It supports different programming approaches. One of the popular approaches to solve a programming problem is by creating objects. This is known as Object-Oriented Programming (OOP).

C# is an object-oriented programming language. All the previous examples are written inside a method inside a class.

class Program
{
    static void Main(string[] args)
    {
        // Code is here
        Console.WriteLine("Hello world!");
    }
}

Static method (or field, property) isn't accessible from an object and is common to the whole class. You cannot create an object of a static class.

Void keyword is a return type if your method doesn't return anything. If a method would return an integer, the keyword would be int.

Access modifiers are used to set the access level/visibility for classes, fields, methods and properties.

There are also two combinations: protected internal and private protected.
